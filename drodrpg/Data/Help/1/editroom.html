Room Editor

The Room Editor is reached by selecting a room in the Editor screen. You can use it to edit any room in the selected hold.

The Room Editor displays a view of the selected room roughly as it would be seen in the game.

Selecting Objects for Placement - A tabbed menu of game objects is shown on the left side of the screen. The objects in DROD are divided into four categories, also called layers:

  1. room pieces: what the room is made from (such as walls, doors, pits, etc)
  2. floor controls: force arrows, ortho tiles, and tile lighting
  3. items: what goes on a tile, such as orbs, scrolls, potions, tar, etc.
    4. and
  4. monsters, NPCs/scripts and level entrance positions.

Select a tab to display that object category. Click on an item in the menu to select it for placement. Many menu items have two or more modes, which may be selected by rotating the tile with the Q/W keys (default), the mouse wheel, or clicking on the menu item again while already selected. Items with several types have a pop-up menu that appears when the item is clicked on again.

Placing Objects - Left-click on a square in the room to place the currently selected item there. Items cannot be placed on a square if something already exists there on the same layer as the object being placed. To delete placed objects, see "Deleting Objects" below. When an item can't or shouldn't be placed on a square, it will be shaded red to indicate a new object won't be placed there.

Setting Object Orientation - For certain objects, their orientation is significant (such as force arrows, monsters, and the player's starting position). Use the rotation keys or the mouse wheel to rotate the way objects get placed. The "Roach" icon on the monster menu tab is updated to indicate which way placed monsters will be oriented. While holding down the left mouse button to place an object, its orientation can be changed until the mouse button is released. When certain customizable items are selected, using the rotation keys or mouse wheel will select the item's mode.

Placing Multiple Objects - Dragging the mouse with the left button down will fill the highlighted region with the selected object on all open squares. Hint: Scenery objects larger than 1x1 tile may be placed this way. Some items, such as scrolls, are intended to be placed only one at a time. For these items, placing more than one at a time is disabled.

Deleting Objects - Right-clicking on a square will delete any object on that square from the currently selected menu layer. Dragging a region with the right mouse button down will delete all objects of that layer in the selected area. Exceptions are checkpoints and individual tile lighting. These may be placed concurrently with other items on their menu tab, so must be explicitly selected in order to delete them. Hold <Shift> while deleting to erase objects from all room layers in the selected region together.

Moving large regions - To make a duplicate copy of a room region, drag the mouse to highlight a rectangular area and press <Ctrl-C> while the mouse button is down to mark the region to be copied. The region marked for copying stays chosen until a new region is marked or you begin editing a different room. Press <Ctrl-V> at any time to re-copy the marked region. Click on a square to paste a copy of that region, the clicked square being the top-left corner. You may use this operation to paste a copied room region into another room within the same level.

Reflecting the room - Press <F7> to reflect the room horizontally (about the vertical axis) and <F8> to reflect the room vertically. All object looks and behaviors are likewise updated to match.

Rotating the room - Press <Ctrl-F7> to rotate the room clockwise. All object appearances and behaviors are likewise updated to match as best as possible.

Undo/Redo - Click the "Undo" button or <Ctrl-Z> to undo changes you've made to the room during this session (since selecting the room for editing). Press "Redo" or <Ctrl-Y> to redo changes that have been undone. Note that undone commands cannot be redone once you starting making other changes to the room.

Editor Settings - Enabling the "Show Errors" checkbox will highlight objects in the room that have an improper or questionable placement (see Room Errors below). Uncheck "Save room automatically" on the Player Settings Screen if you want the editor to prompt you before saving any changes you have made to a room. When the room is saved, if the active player profile is not the author of the room's hold, you will be prompted to make a copy of the hold that your player profile may edit before any room changes are saved.

Playtesting - You can test the current room by hitting F5, then clicking on the room position where you want to begin play. Hit <F4> to define the player's starting stats for playtesting. During playtesting, you may hit <F4> to alter certain game state vars at any time. Playtesting continues until <Escape> is pressed.

Choosing a Room from the Minimap - Click on the level map to select any room in the selected level. You can also click on an empty area adjacent to an existing room to add a new room to the level at that location. Hit <Ctrl-X> to cut and <Ctrl-C> to copy the current room. Hit <Ctrl-V> to paste the marked room to the currently selected spot on the map. Hit the <delete> key to permanently erase a room from the game.

Switching to another level - Press the PageDown and PageUp keys to exit the current room you are working on and go to the next and previous level entrances, respectively.

Customizing and Special Objects - Several items can be customized to look or act a certain way. Customizable objects are the following:

  1. Characters. Special NPC scripts can be defined to look or act a certain way.
  2. Equipment. Items the player may pick up and place in inventory slots.
  3. Orbs. Click on an orb to view which room objects it affects. Gates and lights may be affected. Lightning bolts will flash from the orb to all objects it is defined to act on. Objects highlighted green will be opened/enabled when the orb is struck, objects highlighted orange will be toggled, and objects highlighted red will be closed/disabled. Clicking on an applicable object while in orb editing mode will change the effect the orb has on this object. Clicking on a tile without an orb or affected object will exit orb editing mode. Rotate or click on the orb menu selection again to change the orb type.
  4. Pressure plates. As orbs, except they are activated when depressed. Also, they may be placed as a connected component larger than a single room tile.
  5. Gates. Click on a gate to view which orbs and pressure plates affect it. Lightning bolts will flash from the gate to all objects acting on it. Clicking on an orb or pressure plate while in door editing mode will change the effect the object has on the selected gate. The color coding scheme is the same as for orb/plate customization. Clicking on a tile without an orb, pressure plate or gate will exit door editing mode.
  6. Level entrances. Each level can be entered at any number of user-definable locations. After placing a level entrance, click on it to specify what text the player will see when they enter the level at that point. This text may be the same or different from the descriptions for other level entrances. You may set the entrance text to be shown only the first time the player uses the entrance, or it may be disabled entirely, which will avoid transitioning to the level entrance screen when going to this level entrance. Click on the entrance again to revise the text description, reorient the direction the player faces when entering the level here, to change whether and when the entrance text screen is shown when starting from this entrance, or to mark this entrance as the main entrance to the level. The main level entrance designates its room as the level's Entrance room, or the relative origin from which all other rooms locations in the level are measured. Click and drag a level entrance with the mouse to reposition it elsewhere in the room.
  7. Stairs. After placing stairs, a dialog box will appear with the list of all level entrances in the hold and their text descriptions. Select one of the entrances to designate where the player goes from this staircase. Selecting "End hold" means the player will finish the hold on using these stairs. The destination level can be changed at any time by clicking on the staircase again. Hint: use pairs of stairways and adjacent level entrances to make two-way staircases.
  8. Serpents. Serpents are placed a piece at a time, starting from its head. Once you start placing a serpent, click on a room square to mark a turn in its body. Continue placing segments by clicking to lay down each new piece of the serpent. Pieces that have already been placed can be removed by clicking on part of the last segment placed. To finish placing the monster, click twice on the same spot. A serpent must be at least two squares long. Click on any previously placed serpent to modify it.
  9. Light sources. Click light sources after placement to change their color to the currently selected light color. If their light color already matches the current color selection, clicking lights will increase their light intensity and radius by a tile, up to a maximum of eight tiles. Shift-click lights to decrease their intensity and range.
  10. Custom room textures. Several floor textures are included with each room style. Additionally, you may add your own textures for floors, walls, and pits by selecting color swirl tile on the object menu. When first placing a custom texture tile in a room, you will be prompted to import an image file into your hold from disk to use as your custom texture. These images should be a multiple of the room tile size, which is 44x44 pixels. Only JPEG and PNG images are supported for import, and transparency is not supported. Once one or more images have been imported into your hold, you may select from these for display in any room in the hold. Only one image can be selected as a custom texture for each room. Press F9 to change the image selected for a room and its origin tile (i.e. the coordinates the image starts tiling from).
  11. Scenery obstacles. Scenery obstacles can appear in a variety of different shapes and sizes. You can switch between different types of obstacles with the rotate keys or clicking the scenery menu option again while selected. If you place scenery obstacles by highlighting an area rather than square by square, then the largest possible obstacle of the current type will be placed.   12. Tokens. Architectural tokens may be placed on the item layer to add persistent room effects when touched by the player (with some exceptions, noted below). Rotate or click the token while selected on the object menu to select a different type of token for placement.
    • Arrow rotator (CW/CCW). Rotates all force arrows in the room one-eighth turn clockwise or counter-clockwise when stepped on. Stepping on it again rotates arrows back.
    • Tarstuff toggles. Swaps the nature of two kinds of tarstuff in the room, including monsters of that type.
    • Enhanced vision. Allows the player to see what is under tarstuff.
    • Disarm. Toggles whether the player can use a sword or not. Without a sword, the player does not drop trapdoors. This token does not cancel other situations in which the player is already swordless.

Room Errors - Errors highlighted in the room include the following:

  1. Matching room edges. Room edge squares are highlighted blue if they don't match the squares of an adjacent room. An open square will be highlighted, for instance, if no adjoining room exists, or if a non-matching room element type is at the edge of the adjoining room.
  2. Orbs/plates. Orbs and pressure plates generally affect gates. Orbs that don't affect any objects are marked orange.
  3. Stairs. Stairs that exit the hold are highlighted, not as an error but as a helpful reminder.
  4. Strange placement. If an object is put in a location that might not allow it to be used correctly, it will be marked red.

Misc. Notes -

  1. Here are some notes on good room design.
  2. Drag scrolls, characters, and level entrances to reposition them.
  3. Click on monsters with a different active direction to reorient them.
  4. Press PageUp and PageDown to switch to editing the next and previous levels in the hold.
  5. Tarstuff can be kept off of walls by holding the Shift key when placing.
  6. After modifying a room, realize that saved games recorded previously in that room might no longer be valid.

Editor key command reference -

  • F1 - view these help docs
  • F2 - copy a list of all hold variable references to the clipboard (open a text editor and use the Paste command to display this text)
  • F4 - assign default player stats when playtesting
  • F5 - playtest room
  • F6 - Toggle the display of the in-game appearance of NPCs
  • Ctrl-F7 - rotate room clockwise
  • F7 - reflect room horizontally
  • F8 - reflect room vertically
  • Ctrl-F8 - force reload of the current room graphics. Useful for graphic mod development with the game open.
  • F9 - (re)define a custom room image and its tile origin
  • Ctrl-F9 - (re)define an overhead image and its tile origin
  • PageUp - warp editor to the hold's next logical level entrance
  • PageDown - warp editor to the previous logical level entrance
  • Ctrl-[1-4] - select editor menu tab 1 through 4
  • Ctrl-left - switch to previous editor menu tab
  • Ctrl-right - switch to next editor menu tab
  • Ctrl-Z - undo room edits
  • Ctrl-Y - redo room edits
  • Ctrl-X,C,V - cut/copy/paste a room region highlighted by a mouse drag
  • Alt - hold down to display or hide the in-game appearance of NPCs
  • Enter - pull up CaravelNet chat/console window

Press <Escape> to return to the main editor screen.
